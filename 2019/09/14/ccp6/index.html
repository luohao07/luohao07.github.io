计算机组成原理—计算机编码

September 14, 2019

一 原码，补码，反码

1.1 原码

定义 高最位表示符号位，后面的位表示真值。

例如：原码01101111 = 真值+1101111 原码11101111 = 真值-1101111

原码存在的问题：

  • 零有两种表示法：00000000或10000000
  • 正数+负数时结果不正确：如 1 + (-1) = 000000001 + 10000001 = 10000010 = -2

1.2 补码

1.2.1 补数

补数：以m为模，则一个数b的补数为b模m。所谓的模指的是当一个数超过这个值后就会被溢出。

如若m=12，则9的补数还是9模12=-3，而-9的补数为-9模12=3
例如：生活中的时钟，假如现在时针指向6点，想让时针指向3点的方式有两种：

  • 时针逆时针拨3个小时，即 6 + (- 3) = 3
  • 时针顺时针拨9个小时，即6 + 9 = 15，但是由于时针以12为模，15超过12所以被溢出了，所以实际上得到的结果是：15 模 12 = 3

因此有一个结论[记为结论1]：

加上一个负数 = 加上这个负数的补数。
就像上面说的，在以12为模的情况下：6 + (-3) = 6 + 9 = 3

还有一个结论[记为结论2]：

负数的补数 = 模 + 这个负数。
如 -9的补数 = 12 + (-9) = 3，此处不证明了。

1.2.2 补码

对于一个n = 8位的二进制数来说，它能表示的最大值为1111 1111，我们这里不考虑带符号位，就假定所有的位都表示真值，则它的最大值为$2^8 - 1 = 255$，它的模为2^8 = 256。

一个n位的二进制数的模为 $2^n$

同时我们定义：

一个数的补码为它的补数（即它的真值 + 模）

如：假如一个4位的二进制数，它的模为$2^4 = 10000$，则：

  • $3 = {10000_模 + 0011_真} = 10011_{补}$，超过模的部分溢出：最终为$0011_补$
  • $ - 3 = {10000_{模} + ({-0011}){真}} = 1101{补}$

注意：一个数的补码为真值 + 模，而非 原码 + 模

前面提到原码的一个问题:

正数+负数时结果不正确：如 1 + (-1) = 000000001 + 10000001 = 10000010 = -2

这个问题可以用补码来解决，根据结论1: 加上一个负数 = 加上这个负数的补数:

$ 1 + (-1) $= $1_补+ (2^8 - 1)补$ = ${0000 0001}{补} + {1111 1111}{补}$ = ${0000 0000}{补} = {00000000}_{原} = 0$,

还有一个问题：

零有两种表示法：00000000或10000000

而补码中0只有一个（还是假设n = 4）：

  • $+0 = 10000_{模} + 0000_{真} = 10000_补 = 0000_补$
  • $-0 = 10000_{模} + (-0000_{真}) = 10000_补 = 0000_补$
1.2.3 补码的快捷算法
  • 对于正数$x$：$x_补 = x_真 + 2^n$ 由于 x > 0，所以结果超过了模 $2_n$，最后结果为 $x_真 = x_原$。说明正数的补码为其本身。
  • 对于一个负数$x$：$x_补$ = $x_真 + 2^n$ = $x_真 + 2^n - 1 + 1$
    而$x_真 + 2^n - 1$其实的结果其实就是把x的原码除符号位外其他位数取反，例如：

$x = -3$ 则 原码：${1011}$，真值：${-0011}$
则：$x_真 + 2^n - 1$ = $-0011_真 + 1111 = 1100$

所以$x_补 = x_真 + 2^n - 1 + 1 $ 说明负数的补码为其原码除符号位外其他位取反然后加1

1.3 反码

目前我没有感受到反码存在的意义，它的定义如下：

正数的反码为其原码，负数的补码为其除符号位外其他位取反

唯一的作用，可能就是让我们叙述补码时方便一点：

正数的补码为其原码，负数的原码为其反码 + 1

二 浮点数表示

浮点数的一般表示行式：
$$N=S×r^j$$

  • S：尾数（它写在前面但是它叫尾数），决定了浮点数的精度
  • r：尾数的基数，在计算机中通常取2，4，6，8…
  • j：阶码，它决定了浮点数的上下限

在计算机中的形式：

2.1 IEEE754标准

  • 短实数：也就是计算机中的float
  • 长实数：也就是计算机中的double

如：2.5：=> $10.1_2 = 1.01 × 2^{1}$ ：

  • 数符：正数，所以为0，只有一位，所以二进制还是0
  • 阶码：为1，共有8位，所以二进制是$0000 0001_{原}$，但是注意阶码并不是直接用原码来表示的，而是用原码+127来表示的，所以为应该是:$0000 0001_原 + 0111 1111_{127} = 1000 0000$
  • 尾数：1.01，由于按照科学计数法的要求，个位必需是1，所以这个1可以不保存，直接保存为01，尾数有23位，所以是0100 0000 0000 0000 0000 000

所以短实数2.5的编码是：0 10000000 01000000000000000000000。
