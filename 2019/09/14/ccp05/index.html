计算机组成原理—IO接口

September 14, 2019

一 IO接口的基本组成

  • 设备状态标记用于标记设备是否准备好数据
  • 数据缓冲寄存器DBR用于缓存准备好的数据

二 程序查询方式

2.1 程序查询方式步骤

  • 步骤一：CPU发送命令到IO接口，通知需要从设备中读取数据
  • 步骤二：IO接口从设备中读取数据，同时CPU不断检查IO接口的设备状态标记位。
  • 步骤三：IO接口把设备的数据读取到数据缓冲寄存器后，把设备状态置为就绪
  • 步骤四：CPU检测到数据已经就绪后从数据缓冲寄存器中读取数据。

2.2 程序查询方式的电路

步骤一：启动命令有效，SEL有效，则左则与门生效输出高电平
步骤二：D是设备状态标记位，应该是一个类似与门的东西，经过步骤一后右则为高电平，左则为低电平，所以输出为0，表示还没有准备好数据。
步骤三：B应该是一个异或门电路，经过步骤一后左则为高电平，右则为低电平，所以输出为1，启动设备。
步骤四：设备工作并输入数据到DBR，完成后发送设备工作结束信号。
步骤五：经过步骤四之后，D的左右两则都是高电平，所以它输出为1，表示已经准备好数据。
步骤六：经过步骤四后，B的左右都为高电平，所以他输出0，停止发送启动设备的命令。
步骤七：CPU检测到D输出的高电平后，与IO接口交换数据。

这个电路比较简单，实际上比这复杂很多

三 中断方式

3.0 CPU的中断

待补充，就是最基本的那个中断

3.1 基本电路

  • INTR：中断请求触发器，用于向CPU发送中断请求
  • MASK：中断屏蔽触发器，用于标记设备是否被屏蔽
  • D：IO接口的中断标记位，与上面的程序查询方式类似，受到设备本身的控制

中断流程：

  • 当D输出高电平时，说明设备已经准备好数据，如果该设备没有被屏蔽，则MASK信号也会为高电平
  • MASK高电平 + D高电平 => 中间的与非门（图中的[&]）输出低电平
  • 与非门低电平 => 上面自带高电平的与非门（图中的[1]）输出高电平
  • [1]输出高电平 + CPU查询信号 => 中断请求输出高电平

3.2 链式排队器

$\overline{INTR}$ : 上面3.2的电路图中中断请求信号输出值取反
每个设备包含一个非门和一个与非门

当一个设备没有被中断时：

  • [1]的左则为低电平，右则输出高电平，所以向上的那根信号线INTP输出为1
  • $\overline{INTR}$输出高电平，所以与非门[&]输出为0

当一个设备被中断时：

  • [1]的左则为低电平，右则输出高电平，所以向INTP还是输出1
  • $\overline{INTR}$输出低电平，则与非门[&]输出为高电平
  • 一量[&]输出高电平，则下一个设备的[1]输出为低电平，则下一个设备INTP输出为低电平。
    所以当一个设备被中断时，它以它前面的设备INTP输出都是高电平，而它之后的设备INTP都输出了低电平。即当链式排队器输出为1111111000000时，最后一个1是所有输出了中断信号的设备中优先级最高的。

查找被中断的设备中优先级最高的：

把INTP信号与输入的INTR(注意不取反)做与操作，假如排队器输出的内容是11110000，则：

  • 前面三个1虽然INTP输出为1，但是INTR为0，所以&操作后的值还是0而
  • 后面4个0INTP输出的值就是0，所以输出还是0
  • 最后的那一个1，INTP输出为1，且INTR也为1，所以输出为1
  • 因此得到的结果就是00010000

3.3 找到中断程序入口


几个概念：

  • 中断向量：保存中断信息的寄存器的集合，寄存器里面保存了中断程序地址的跳转信息等数据
  • 向量地址：中断向量寄存器的位置
  • 中断向量地址形成部件，用于把排队器输入的信号（如上面说的00010000）转成向量地址

步骤：

  • 步骤一：排队器输入信号，中断向量地址部件输出向量地址
  • 步骤二：CPU读取向量地址的信息，如12H，则找到内存中的12H的数量
  • 步骤三：CPU根据向量地址的内容跳转，例如12H中保存的数据是JMP 200，则跳转到200，这个位置就是中断程序的开始地址。
