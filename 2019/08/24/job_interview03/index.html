面试重点 — mysql和redis

August 24, 2019

redis

重点知识

  • redis支持的数据类型
  • redis的缓存淘汰机制
  • redis 备份方式以及实现原现
  • redis主从模式的实现方式
  • redis集群（多主）的实现方式

面试题：

  • 你知识redis中支持哪些数据结构吗
  • redis中缓存的淘汰机制有哪些
  • redis的复制方式有哪些，是怎么实现的

推荐看书：

  • 《redis入门指南》入门级的书，估计三四个小时就能看完了，看完之后应对面试应该没什么问题
  • 《redis设计与实现》建议看完《redis入门指南》再看，讲得稍微深入一点，系统全面一点

mysql

重点内容：

  • 关联查询，左右，右外，全外和内关联的作用及区别
  • 对group by 语句的使用，笔试中经常会考到
  • innodb中的索引结构，B+树结构，以及索引为什么要用B+树，聚集索引和非聚集索引的区别
  • mysql查询优化的内容
  • mysql事务隔离级别，脏读，幻读，不可重复读问题
  • mysql主从复制原理，涉及到哪些线程

笔试题：
有一张User表如下，请删除名称重复的数据，即同名的数据只保留一条
id|name|sex
-|-|-
1|张三|F
2|张三|F
3|李四|M
4|李四|M
5|李四|M
6|王五|F

这里考的是对group by 语句的使用

DELETE FROM user WHERE id NOT IN (SELECT max(id) FROM User GROUP BY name)

面试题：

  • mysql的索引结构是什么，聚集索引和非聚集索引有什么区别，为什么要用B+树（非常重要，10次面试有8次会问到）
  • mysql的事务隔离级别有哪些？（这种问题最好不要只回答，有XXXX四种，还要讲一下每一种的特点）
  • mysql主从复制原理，涉及到哪些线程

推荐看书：

  • 《mysql必知必会》：一些基础知识，看完之后并不一定能应对面试
  • 《mysql技术内幕innodb存储引擎》：这本书写得非常深，关键看下索引的那一章，想全部看完应该非常要时间，但是如果全看完看懂，那mysql这一块可以算是大师级的人物了
